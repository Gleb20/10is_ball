# PRD — Tab-10 MVP

Статус: Implementation-ready v2.0  
Подход: TDD mandatory

## 1. Scope

Документ описывает автономный закрытый продукт с локальными аккаунтами и локальной теннисной админкой.

## 2. Акторы и права

### Системные роли
- `admin` — все возможности `user` плюс управление локальными аккаунтами.
- `user` — все спортивные возможности.

### Контекстные права
- организатор матча;
- организатор турнира;
- текущий судья;
- капитан команды;
- участник события.

### Общие правила
- Любой активный `user` может создать матч, турнир или команду.
- Только капитан приглашает в команду, редактирует её и исключает участников.
- Любой участник команды может без подтверждения включить другого текущего участника этой команды в матч или турнир.
- `admin` не получает права менять завершённые спортивные результаты.

## 3. AUTH — локальный вход

### AUTH-001 Закрытый доступ
Без активной сессии доступны только экран входа. Все продуктовые URL ведут на логин.

### AUTH-002 Создание аккаунта
Самостоятельной регистрации нет. Аккаунт создаёт администратор.

Обязательные поля:
- имя;
- фамилия;
- email.

Опциональные:
- дата рождения;
- организация;
- должность.

### AUTH-003 Временный пароль
При создании/сбросе аккаунта администратор получает случайный временный пароль, отображаемый один раз. При первом входе пользователь обязан задать новый пароль.

### AUTH-004 Пароль
Новый пароль:
- не короче 10 символов;
- содержит цифру;
- строчную латинскую букву;
- заглавную латинскую букву;
- допустимый спецсимвол.

Поле скрыто, имеет действие «показать». Генератор пароля разрешён.

### AUTH-005 Обычный вход
Email + пароль. Заблокированный пользователь не входит.

### AUTH-006 Сессия
- срок сессии — 7 дней с момента последней успешной активности;
- несколько обычных сессий пользователя разрешены;
- logout завершает текущую сессию;
- смена пароля сохраняет текущую сессию и завершает остальные.

### AUTH-007 Восстановление
Self-service recovery и email-отправка не входят в MVP. Администратор сбрасывает пароль и выдаёт новый временный пароль.

### AUTH-008 Активные сессии
В профиле отображаются активные сессии пользователя; можно завершить другую сессию.

### TDD gate
До реализации написать тесты `AT-AUTH-001..012`.

## 4. ADM — теннисная админка

### ADM-001 Доступ
Админка доступна только роли `admin`.

### ADM-002 Список пользователей
Показывает имя, email, роль, статус, дату создания, дату последнего входа. Есть поиск по имени/email и фильтр статуса.

### ADM-003 Создание
Администратор создаёт пользователя, выбирает `user` или `admin`, получает временный пароль.

### ADM-004 Редактирование
Можно менять имя, фамилию, дату рождения, организацию, должность и роль. Email в UI MVP не меняется.

### ADM-005 Блокировка
Блокировка немедленно завершает все сессии. Пользователь исчезает из выбора для новых событий, но остаётся в истории.

### ADM-006 Разблокировка
Возвращает возможность входа и участия в новых событиях.

### ADM-007 Сброс пароля
Создаёт новый временный пароль, завершает все сессии и требует смены пароля при следующем входе.

### ADM-008 Аудит
Фиксируются создание, редактирование, блокировка, разблокировка, смена роли и сброс пароля.

### TDD gate
До реализации написать тесты `AT-ADM-001..010`.

## 5. HOME — главная

### HOME-001 Состав
- avatar/statistics hero;
- CTA «Начать»;
- активные события;
- последние 5 матчей/турниров;
- топ-3 рейтинга;
- вход в уведомления;
- вход в профиль.

### HOME-002 Hero
Показывает:
- сыграно матчей;
- победы;
- поражения;
- win rate;
- средние очки за матч;
- принципиального соперника с CTA «Реванш».

### HOME-003 Последние события
Карточка матча: стороны, итог, победитель, длительность, формат, судья/роль текущего пользователя.  
Карточка турнира: топ-3, длительность, отметка участия или судейства.

### HOME-004 Активные события
Можно одновременно показывать турнир и один отдельный матч. Пользователь не может играть два активных матча одновременно.

### HOME-005 Рейтинг
По умолчанию топ-3 за всё время; быстрый переключатель «за всё время / за месяц».

### HOME-006 Empty states
Для отсутствующих матчей, рейтинга, соперника и активных событий показывается объяснение и релевантный CTA.

## 6. PROFILE — профиль

### PROFILE-001 Свой профиль
Показывает:
- имя, фамилию, email, организацию, должность, дату рождения, аватар;
- матчи, победы, поражения, win rate, средние очки;
- турниры, победы в турнирах, созданные турниры;
- матчи в роли судьи;
- текущий общий рейтинг;
- места в текущих командах.

### PROFILE-002 Интересные факты
- самая долгая игра;
- лучший победный счёт;
- частый соперник;
- принципиальный соперник.

Частый и принципиальный соперник считаются только среди зарегистрированных активных или исторических пользователей, не гостей.

### PROFILE-003 Редактирование
Можно изменить локальные поля профиля и аватар. Email не меняется.

### PROFILE-004 Аватар
При создании аккаунта случайно назначается один заранее подготовленный плейсхолдер. Повторы допустимы. Пользователь может:
- получить новый случайный плейсхолдер;
- загрузить свою картинку.

Ручного выбора из каталога нет.

### PROFILE-005 Чужая карточка
Доступна всем активным пользователям и показывает:
- имя, фамилию, аватар, организацию, должность;
- матчи, победы, поражения, win rate, средние очки;
- сыгранные турниры, победы в турнирах, созданные турниры;
- матчи в роли судьи;
- место в рейтинге;
- CTA «Бросить вызов».

Дата рождения и email не показываются.

### PROFILE-006 Актуальные данные в истории
После изменения имени или аватара новые данные отображаются и в исторических карточках.

## 7. RANK — рейтинг

### RANK-001 Формула
Рейтинг строится по абсолютному количеству побед.

Порядок:
1. `wins DESC`;
2. `win_rate DESC`;
3. `matches_played DESC`;
4. `created_at DESC` — позже созданный аккаунт выше.

### RANK-002 Периоды
- all time — по умолчанию;
- текущая календарная неделя;
- текущий календарный месяц.

### RANK-003 Матчи с гостями
Учитываются в рейтинге зарегистрированного игрока. Гость в рейтинг не попадает.

### RANK-004 Командный рейтинг
Сравниваются текущие участники команды по их общим показателям.

### RANK-005 Карточка игрока
Нажатие открывает публичную карточку.

## 8. HISTORY — история

### HISTORY-001 Состав
Единый список матчей и турниров пользователя.

### HISTORY-002 Фильтры
- период;
- роль игрок/судья;
- победа/поражение;
- матч/турнир;
- поиск по имени соперника или названию турнира.

### HISTORY-003 Видимость
- активные события видят только участники, судьи и организатор;
- завершённые события видят все активные пользователи.

### HISTORY-004 Неактивные аккаунты
Заблокированный пользователь скрыт из новых списков и рейтинга, но остаётся в исторических событиях.

## 9. MATCH — создание и жизненный цикл матча

### MATCH-001 Создание
Организатор всегда является игроком. Название по умолчанию: `Матч {дата и время}`. Название меняется только до старта.

### MATCH-002 Формат
- `1v1`;
- `2v2` только для обычного матча.

### MATCH-003 Участники
- зарегистрированные пользователи;
- участники общей команды без подтверждения;
- одноразовые гости: имя + фамилия.

Гость создаётся заново для каждого события и не имеет профиля.

### MATCH-004 Быстрый выбор
Селектор показывает:
- поиск по имени/фамилии;
- мои команды;
- недавние соперники;
- частые соперники.

### MATCH-005 Правила
Обязательная настройка до старта:
- очки до победы: 11, 21 или своё значение;
- сухая победа: включена/выключена и порог;
- определение первого подающего: random/manual/rally;
- формат 1v1/2v2.

После старта правила неизменны.

### MATCH-006 Победа
Обычная победа: достигнут лимит и отрыв не менее 2. При разнице 1 игра продолжается; после достижения порога подача переходит после каждого очка.

Сухая победа: при включённом правиле матч завершается при достижении настроенного отрыва.

### MATCH-007 Подача
До deuce смена после двух подач, после deuce — после каждой. В 2v2 система знает текущего подающего внутри пары и поддерживает упрощённую ротацию и смену сторон.

### MATCH-008 Старт
Матч стартует только при валидном составе:
- 1v1 — две стороны по одному игроку;
- 2v2 — две стороны по два игрока.

### MATCH-009 Конкурентность игрока
Пользователь не может одновременно играть в двух активных матчах. Участие в турнире как сущности не блокирует отдельный матч, если текущий турнирный матч пользователя не активен.

### MATCH-010 Завершение
После победного очка время останавливается, но результат становится окончательным только после подтверждения судьёй. До подтверждения можно отменить окончание и продолжить.

### MATCH-011 Read-only
После подтверждения результат, правила и логи недоступны для редактирования.

### MATCH-012 Остановка
Организатор или текущий судья выбирает:
- причину из трёх пресетов или свой текст;
- победителя.

Остановленный матч учитывается в истории, статистике и рейтинге.

### MATCH-013 Неявка
Организатор или текущий судья вручную фиксирует неявку. Вторая сторона получает победу; причина видна в деталях.

### MATCH-014 Быстрый вызов
«Бросить вызов» и «Реванш» открывают предзаполненный flow создания матча. Можно подтвердить правила по умолчанию или изменить их.

### MATCH-015 Черновики
Незавершённый flow создания не сохраняется.

## 10. JUDGE — судейство

### JUDGE-001 Один судья
У матча может быть только одна активная judge session.

### JUDGE-002 Допустимые судьи
- участник обычного матча;
- участник турнира;
- зарегистрированный пользователь, приглашённый судить турнир/матч.

### JUDGE-003 Захват
Первый допустимый пользователь, успешно захвативший свободный слот, становится судьёй.

### JUDGE-004 Устройство
Judge session привязана к `user_id + auth_session_id`. Один пользователь не может судить с двух устройств и не может судить два матча одновременно.

### JUDGE-005 Занятый слот
Другому пользователю показывается:
> Этот матч уже судит «Имя пользователя», два судьи у матча — дело к драке. Давай не будем.

### JUDGE-006 Release
Судья может освободить слот. Матч остаётся `in_progress` и показывает, что требуется судья.

### JUDGE-007 Handover
Судья может атомарно передать слот конкретному допустимому пользователю. До принятия/старта новой judge session слот зарезервирован за ним.

### JUDGE-008 Heartbeat
Клиент отправляет heartbeat. После заданного TTL без heartbeat judge session освобождается. Рекомендация: heartbeat 30 секунд, TTL 120 секунд.

### JUDGE-009 Начисление
Каждое очко отправляется на сервер с idempotency key и expected match version.

### JUDGE-010 Undo
Undo откатывает последнее действующее очко и всё производное состояние: счёт, подачу, deuce, потенциальное завершение.

### JUDGE-011 Аварийная коррекция
Спрятана в дополнительном меню. Разрешена только активному судье до подтверждения результата. Коррекция оставляет технический audit trail.

### JUDGE-012 Чистый пользовательский лог
В отображаемом логе остаются только действующие очки. Отменённые события сохраняются технически, но не показываются как часть чистой истории.

## 11. TOURNAMENT — турнир

### TOURNAMENT-001 Создание
Название по умолчанию: `Турнир {дата и время}`. Название меняется до старта.

Организатор до приглашения выбирает:
- участвует как игрок;
- только организует.

### TOURNAMENT-002 Минимум/максимум
- минимум 3 игровых участника;
- максимум 64;
- организатор без участия не считается игроком.

Если игровых участников двое, создаётся обычный матч, а не турнир.

### TOURNAMENT-003 Форматы
- single elimination;
- double elimination с одним финальным матчем.

Round robin вне MVP.

### TOURNAMENT-004 Матчи
Только 1v1. Один набор правил применяется ко всем матчам турнира.

### TOURNAMENT-005 Сбор участников
Источники:
- приглашённые зарегистрированные пользователи;
- текущие участники команд;
- гости.

Быстрое приглашение зарегистрированного пользователя живёт 10 минут. Турнир не стартует автоматически.

### TOURNAMENT-006 Комната ожидания
Показывает статус сбора, подтверждённых/автоматически добавленных участников, ожидающие приглашения и CTA «Сгенерировать сетку» при 3+ игроках.

### TOURNAMENT-007 Генерация сетки
После нажатия:
- состав закрывается;
- pending приглашения становятся неактуальными;
- строится сетка;
- организатор может менять пары и bye-позиции.

### TOURNAMENT-008 Посев
Источник — рейтинг all time. Игроки без рейтинга и гости распределяются случайно среди несеяных.

### TOURNAMENT-009 Bye
Сетка допускает нечётный состав. Система назначает проходы автоматически; организатор может переставить участников до старта. Один игрок не должен получать больше одного bye за турнир, если это математически возможно.

### TOURNAMENT-010 Роспуск сетки
До старта организатор может распустить сетку и вернуться к сбору участников. Принятые участники сохраняются.

### TOURNAMENT-011 Выход до старта
Участник может выйти. Если сетка уже построена, она получает статус `needs_regeneration` и не может быть запущена до новой генерации.

### TOURNAMENT-012 Старт
«Начать турнир» фиксирует сетку. После старта нельзя добавлять участников, менять пары, правила или формат.

### TOURNAMENT-013 Параллельность
Разрешены параллельные матчи, если один игрок и один судья не заняты одновременно в нескольких активных матчах.

### TOURNAMENT-014 Неявка/выход после старта
Ручная фиксация даёт техническое поражение в текущей паре и продвигает соперника.

### TOURNAMENT-015 Завершение
Формируются:
- места;
- сумма игровых очков по всем матчам;
- количество сыгранных матчей;
- топ-3.

### TOURNAMENT-016 Остановка
Остановить может организатор. Указывается причина. Сыгранные матчи учитываются в статистике и рейтинге; несыгранные отменяются; победитель турнира и итоговый топ не формируются.

### TOURNAMENT-017 Детальная карточка
Показывает сетку, длительность, выделение линии текущего пользователя, активный матч, следующий матч и список итогов. Нажатие на матч открывает его детали.

### TOURNAMENT-018 Отмена до старта
Организатор может отменить турнир в статусах `collecting` / `bracket_generated` / `needs_regeneration`. Турнир переходит в `cancelled`, сетка сбрасывается, pending-приглашения помечаются cancelled. В истории остаётся как «Отменён», а не «вечный сбор».

## 12. TEAMS — команды

### TEAM-001 Создание
Любой пользователь может создать неограниченное число команд и состоять в нескольких. Обязательное поле — название; аватар, слоган и welcome text опциональны.

### TEAM-002 Капитан
Один капитан. Создатель становится капитаном.

### TEAM-003 Права капитана
- приглашать в команду;
- менять название, аватар, слоган, welcome text;
- исключать участников;
- передавать капитанство.

### TEAM-004 Приглашение
- TTL 14 дней;
- показывается в notification center;
- активное приглашение можно принять/отклонить;
- повторное приглашение — новая запись.

Email не используется.

### TEAM-005 Принятие
После принятия показывается экран приветствия команды.

### TEAM-006 Выход
Обычный участник выходит самостоятельно. Капитан сначала передаёт капитанство.

### TEAM-007 Автопередача
Если капитан заблокирован или удалён из команды без ручной передачи, капитаном становится текущий участник с самой ранней датой вступления. Он получает уведомление с игровым текстом.

### TEAM-008 Архив
Если участников не осталось, команда архивируется и исчезает из активных списков, сохраняя историю.

### TEAM-009 История
Выход не переписывает исторические матчи и турниры. Командный рейтинг строится по текущему составу.

## 13. NOTIF — уведомления

### NOTIF-001 Центр уведомлений
Отдельный экран, доступный с главной и из профиля. Карточка содержит дату/время, сообщение, статус и действия.

### NOTIF-002 Типы MVP
- приглашение в матч;
- приглашение в турнир;
- приглашение судить;
- приглашение в команду;
- добавление в турнир командой;
- остановка матча/турнира;
- передача судейства;
- назначение капитаном;
- системные изменения доступа к аккаунту.

### NOTIF-003 Popup
Если пользователь находится в активной сессии и приглашение актуально, можно показать одну ненавязчивую шторку/диалог. Просроченное приглашение при следующем входе popup не вызывает.

### NOTIF-004 История
Устаревшая карточка остаётся и показывает причину: timeout, событие началось, отменено, завершено, состав закрыт.

### NOTIF-005 Read state
Хранится read/unread, но без агрессивных бейджей. Допустим один спокойный индикатор в точке входа.

### NOTIF-006 TTL
- матч/турнир/судейство: 10 минут;
- команда: 14 дней.

## 14. ONB — онбординг

### ONB-001 Первый запуск
Автоматически запускается один раз после первого успешного входа и смены временного пароля.

### ONB-002 Возможности
- подсветить главную, рейтинг, историю, уведомления, профиль и CTA «Начать»;
- каждый шаг можно пропустить;
- весь onboarding можно закрыть.

### ONB-003 Учебный матч
Пользователь создаёт учебный матч с системным участником «Призрачный Олег» и проходит judge flow.

### ONB-004 Изоляция
Учебное событие не влияет на историю, статистику, рейтинг и соперников.

### ONB-005 Повтор
Повторный запуск доступен из профиля.

## 17. AUDIT — облегчённый аудит

Фиксировать только критичные действия:
- создание/изменение/блокировка пользователя;
- создание/старт/завершение/остановка матча;
- judge acquire/release/handover;
- создание турнира, генерация/роспуск/старт/остановка сетки;
- завершение турнирного матча;
- создание/изменение команды, membership и капитанство;
- invitation response/expiry;
- аварийная коррекция счёта.

Каждое очко хранится в match event log, а не дублируется в общем audit log.

## 18. Definition of Done для требования

Требование завершено, только если:
- есть тесты до реализации;
- тесты проходят локально и в CI;
- покрыты happy path, error path и минимум один boundary case;
- API и UX состояния согласованы;
- миграции обратимо применяются на тестовой БД;
- нет нарушения read-only и permission rules;
- traceability matrix обновлена;
- нет flaky-тестов.
