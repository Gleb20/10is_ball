# UX Flows — Tab-10 MVP

## 1. Навигация mobile-first

Канон bottom bar (ADR [`D5`](../DECISIONS.md)) — ровно пять primary tabs:

| Tab | Роль |
|-----|------|
| Главная | Home, hero, активные события, топ-3, вход в уведомления |
| История | Лента матчей и турниров; поиск/фильтры в bottom sheet |
| «Начать» | Hub: Матч · Турнир · Challenge / Revenge |
| Рейтинг | All-time / неделя / месяц (+ фильтр по команде) |
| Профиль | Карточка пользователя, сессии, команды, помощь, выход |

Правила:
- Уведомления — из главной и профиля (не отдельный tab).
- Админка — отдельный пункт только для `admin` (профиль / CTA на главной), не в bottom bar.
- Списки матчей и турниров — secondary routes из «Начать», Истории и карточек Home; **не** primary tabs.
- Judge mode и auth-экраны — вне tab shell (без bottom nav).

> Реализация до Phase 10 может ещё показывать устаревшие табы «Матчи»/«Турниры»; целевое состояние — таблица выше.

## 2. Первый вход

1. Администратор создаёт аккаунт и передаёт временный пароль.
2. Пользователь вводит email и временный пароль.
3. Система требует создать новый пароль и повторить его.
4. После успешной смены открывается одноразовый онбординг.
5. Пользователь может пройти учебный матч с Призрачным Олегом или пропустить.
6. Открывается главная с корректными empty states.

Ошибки:
- неверная пара email/пароль;
- аккаунт заблокирован;
- временный пароль уже заменён;
- новый пароль не соответствует правилам;
- повтор не совпадает.

## 3. Обычный вход

1. Email.
2. Пароль.
3. «Войти».
4. При успехе — главная.
5. При необходимости «Показать пароль».

Self-service «Забыли пароль?» не показывается. Вместо него текст: «Обратитесь к администратору Tab-10».

## 4. Админ создаёт пользователя

1. Админка → Пользователи → «Добавить».
2. Ввод обязательных и опциональных полей.
3. Выбор роли `user/admin`.
4. Подтверждение.
5. Система показывает временный пароль один раз с CTA «Скопировать».
6. Пользователь появляется в списке.

## 5. Создание матча

1. «Начать» → «Матч» (route создания: `/matches/new`, ADR D5 / Q-UI-2).
2. Название по умолчанию, возможность изменить.
3. Выбор 1v1/2v2.
4. Настройка правил.
5. Выбор зарегистрированных пользователей или добавление гостя.
6. Опциональное приглашение судьи.
7. Проверка состава.
8. Старт / ожидание принятия внешнего приглашения.
9. Переход в карточку активного матча.

Для участника команды подтверждение не требуется.

## 6. Challenge / Revenge

1. CTA из профиля, рейтинга или hero.
2. Открывается тот же setup матча.
3. Соперники предзаполнены.
4. «Быстрый старт» с дефолтами либо раскрытие настроек.

## 7. Judge mode

1. Карточка активного матча → «Стать судьёй».
2. Сервер атомарно захватывает слот.
3. При успехе — judge UI; при конфликте — дружелюбная ошибка с именем текущего судьи.
4. На мобильном предлагается горизонтальная ориентация.
5. Судья расставляет стороны и определяет первую подачу.
6. Крупные зоны сторон начисляют очко.
7. UI показывает текущего подающего.
8. Undo откатывает последнее очко и подачу.
9. Дополнительное меню содержит аварийную коррекцию, release, handover, stop.
10. После победного условия показывается экран предварительного результата.
11. «Подтвердить» делает результат read-only; «Продолжить» откатывает победное завершение.

При разрыве сети:
- неподтверждённое локальное действие не считается сохранённым;
- после восстановления клиент получает серверное состояние;
- повторная отправка с тем же ключом не создаёт второе очко.

## 8. Создание турнира

1. «Начать» → «Турнир».
2. Организатор выбирает, играет ли сам.
3. Название, формат сетки, единые правила.
4. Добавление игроков и гостей.
5. Комната ожидания.
6. При 3+ игроках активна «Сгенерировать сетку».
7. После генерации состав закрывается, приглашения устаревают.
8. Организатор редактирует пары/bye.
9. «Начать турнир» фиксирует сетку или «Распустить сетку» возвращает к сбору.
10. Активная сетка показывает текущий и следующий матч.
11. По завершении всех матчей показывается итог.

## 9. Выход из турнира

- До генерации: удаление из состава.
- После генерации, до старта: предупреждение; сетка становится невалидной.
- После старта: подтверждение технического поражения.

## 10. История и детали

1. Вкладка «История» — основной вход в ленту матчей и турниров (ADR D5).
2. С главной: блок «последние события» и переход «все события» → та же История.
3. Поиск и фильтры открываются в bottom sheet.
4. Нажатие на матч показывает итог и чистый лог.
5. Нажатие на турнир показывает сетку и результаты.
6. Нажатие на участника открывает публичную карточку, если это зарегистрированный пользователь.

## 11. Рейтинг

1. Главная: топ-3 all time, переключатель месяц.
2. «Весь рейтинг».
3. Фильтр all time / неделя / месяц.
4. Фильтр общая выборка / конкретная моя команда.
5. Нажатие на игрока → карточка → «Бросить вызов».

## 12. Команды

### Создание
1. Профиль → Команды → «Создать».
2. Название, опциональные аватар/слоган/welcome text.
3. Создатель становится капитаном.

### Приглашение
1. Капитан выбирает пользователя.
2. У приглашённого появляется notification card.
3. Accept → welcome screen; Decline → подтверждающий snackbar.
4. Через 14 дней карточка становится устаревшей.

### Выход и капитанство
- обычный участник подтверждает выход;
- капитан сначала выбирает нового капитана;
- при автоматической замене новый капитан получает уведомление.

## 13. Notification center

- активные actionable карточки сверху;
- обработанные и устаревшие ниже по времени;
- просроченная карточка объясняет причину;
- popup только для актуального приглашения в текущей активной сессии;
- открытие списка помечает видимые карточки прочитанными.

## 14. Empty states

Каждое пустое состояние содержит конкретный следующий шаг:
- нет матчей → «Создать первый матч»;
- нет рейтинга → «Сыграть матч»;
- нет команд → «Создать команду»;
- нет уведомлений → нейтральный текст без CTA;
- нет соперника → «Проведи несколько матчей»;
- пустая админка → «Добавить пользователя».

## 15. Responsive behavior

- Mobile (от 360 px) — основная композиция; shell max-width ~480 px на узких экранах (ADR D5 / Q-UI-3).
- ≥768 px — можно расширить контентную колонку и параллельные блоки, **без** смены набора и порядка bottom tabs и без смены порядка действий.
- Judge mode — landscape на мобильном; bottom nav скрыт.
- Турнирная сетка — горизонтальный scroll/zoom, фиксированные подписи раундов.
- Desktop не вводит отдельную IA и не добавляет primary destinations вне ADR D5.
